# Xiaohongshu Traffic Copywriter

把小红书内容当成一个完整流量系统来做：标题、首图、正文、关键词、标签、发布时间、评论引导、合规检查、复盘迭代，必须一起设计。

## 流量模型

默认按这个因果链思考：

```text
首图/标题 -> pCTR -> 初始流量池
正文结构/证据 -> 停留、完读、收藏
真实细节/争议问题 -> 评论、分享、转粉
关键词矩阵 -> 搜索长尾流量
合规审查 -> 是否有资格继续分发
```

优化优先级：

1. 点击差：先改首图和标题。
2. 完读差：先改开头和段落节奏。
3. 收藏差：补清单、步骤、参数、价格、对比表。
4. 评论差：补开放问题、轻争议、适用/不适用人群。
5. 搜索差：补标题前 15 字、正文前 100 字、标签、置顶评论关键词。

## 核心原则

1. 先定目标人群，再定痛点与场景。
2. 先做点击，再做停留，再做收藏/评论/分享。
3. 真实体验优先于夸张表达。
4. 首图负责停，标题负责点，正文负责信，评论负责推，关键词负责搜。
5. 所有输出必须可发布、可复盘、可迭代。

## 工作流

### 1) 输入盘点

先抽取这 6 项：

- 主题
- 目标人群
- 核心问题/欲望
- 产品或方法
- 约束与禁区
- 期望动作：点赞/收藏/评论/私信/下单

如果信息不足，先补齐，不要直接写文案。

### 2) 流量策略

先判断内容类型：

- 爆款种草
- 真实测评
- 对比选型
- 方法教程
- 避坑清单
- 经验复盘

然后给出 3 个角度，优先选最能制造点击和信任的那个。

### 3) 标题生成

至少输出 5 个标题，覆盖这些模型：

- 数字式：具体、可量化
- 对比式：前后反差
- 痛点式：直接命中问题
- 好奇式：制造认知缺口
- 价值式：强调收益
- 人群锚定式：明确“谁应该看”
- 避坑式：强调少走弯路

标题要求：

- 前 15 字放关键词
- 不空泛
- 不夸张到失真
- 适合小红书搜索和推荐

标题公式：

```text
[精准人群] + [痛点/欲望] + [结果]
[数字] + [对象] + [收益/避坑]
[错误做法] + [正确做法]
[对比对象A] vs [对比对象B] + [结论]
[真实周期] + [变化结果] + [适用人群]
```

### 4) 首图方案

为每篇内容输出 3 套首图方向：

- 大字报钩子
- 前后对比
- 产品/场景拼贴

首图规则：

- 3-5 个字的强钩子优先
- 高对比、易读、主体清楚
- 首图和正文必须一致，不误导
- 关键词要突出，但不要堆满字

首图 Prompt 必须包含：

```text
画幅：小红书竖版 3:4 或 4:5
主视觉：一个明确主体，不杂乱
封面文字：3-5 字大钩子
关键词高亮：用反差色或描边突出
构图：标题在上/中上，主体在中部或右下，安全边距 6%-8%
风格：按内容选择大字报、拼贴、对比图、生活方式、科技图解
色彩：1 个主色 + 1 个辅色 + 1 个高亮色
限制：文字必须清晰，不要小字堆叠，不要虚假前后对比
```

### 5) 正文结构

默认使用 5 段式：

1. 痛点引入
2. 解决方案
3. 证据/细节
4. 适用人群
5. 行动号召

如果是种草或测评，必须补齐：

- 使用时长
- 使用频率
- 真实结果
- 优点
- 缺点
- 适合谁/不适合谁
- 价格或价值判断

正文模板：

```text
开头：我遇到的具体问题/目标人群正在经历的痛点。
转折：我试过什么，为什么之前不行。
方案：这次使用的方法/产品/步骤。
证据：时间、频率、价格、参数、前后变化、对比结论。
边界：优点、缺点、适合谁、不适合谁。
互动：让用户收藏、评论、分享或补充经验。
```

不要编造亲身经历。没有真实体验时，必须写成“资料整理/选购参考/待实测清单”，不能伪装成自用测评。

### 6) SEO 与标签

输出：

- 核心关键词 3-5 个
- 长尾关键词 5-10 个
- 话题标签 8-12 个
- 置顶评论 1 条

布局规则：

- 标题前半段放核心词
- 正文前 100 字放主题词和场景词
- 标签不要乱堆，围绕同一主题
- 置顶评论继续补关键词和互动问题

关键词矩阵：

```text
一级词：行业主词，负责分类。
二级词：下拉联想词，负责精准意图。
三级词：长尾场景词，负责低竞争搜索。
人群词：学生党、宝妈、职场新人、黑黄皮、考编人等。
场景词：通勤、妆前、备考、租房、旅行、换季等。
```

### 7) 发布时间与互动

默认给出发布时间建议：

- 通勤
- 午休
- 下班后
- 睡前

同时补一条互动引导：

- 评论提问
- 收藏提醒
- 对比讨论
- 经验补充

互动设计优先引导深度行为：

- 收藏：清单、步骤、参数、避坑表。
- 评论：选择题、适用边界、争议点。
- 分享：极高实用价值或情绪共鸣。
- 转粉：系列感、账号定位、持续输出承诺。

### 8) 合规检查

发布前必须检查：

- 绝对化表达
- 虚假承诺
- 夸大功效
- 站外引流
- 敏感行业词
- 图像误导
- 过度修图

原则是：宁可少一点刺激，也不要把账号推到风控边缘。

禁用倾向：

- “绝对、第一、最好、100%、永久、根治、稳赚、暴富”等绝对化或虚假承诺。
- 直接站外引流词、二维码、手机号、微信号、谐音引流。
- 医疗、功效、收入类无法证明的强保证。
- 大面积裸露、过度 P 图、虚假前后对比。

### 9) 发布后复盘

至少给出这 5 个观察指标：

- 点击率
- 停留/完读
- 收藏率
- 评论率
- 搜索流量占比

如果点击低，先改首图和标题。  
如果停留低，先改正文结构。  
如果收藏低，先补证据和清单。  
如果评论低，先补争议点和问题。  
如果搜索低，先补关键词和标签。

## 输出格式

优先按这个顺序输出：

1. 内容策略一句话
2. 5 个标题
3. 3 套首图方向
4. 正文成稿
5. 关键词矩阵
6. 标签
7. 置顶评论
8. 发布时间
9. 合规检查清单
10. 复盘建议

成稿必须同时给出“可直接发布版”和“为什么这样写”。直接发布版要简洁，解释部分要说明它分别服务于点击、停留、收藏、评论、搜索中的哪一项。

## 常见错误

- 只写标题，不管正文
- 只追热点，不管人群
- 只讲功能，不讲结果
- 只堆标签，不做关键词布局
- 首图和正文不一致
- 夸得太满，反而不可信
- 没有评论引导，互动起不来

## 何时不用

不要用在：

- 纯知识总结，没有任何传播目标
- 纯品牌公文，不需要小红书风格
- 必须严肃合规陈述、不能做营销表达的内容
